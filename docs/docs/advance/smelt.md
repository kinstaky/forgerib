# 重组熔炼

XIA 和 VME 中的事件是不同的：XIA 中的事件是单个通道的，所以需要以触发为脊，将和触发同属一个物理事件的探测器事件填到对应的触发事件中；VME 中的事件已经打包，但是不是所有的触发都包含了 VME 触发，所以只需要将 VME 事件填到有 VME 触发的触发事件中即可。

事件重组的前提是已经有组织好的触发事件，至少有：

+ XIA 的时间戳，用于链接其它的 XIA 的事件；
+ VME 的 entry，用于链接 VME 事件。

在具体实现是，使用了长度为 1000 的事件缓存，并用 `tofill` 标记下一个填充的事件编号（初始为0）。程序的大致实现为

+ 读入所有的 XIA 触发
+ 然后逐个读取 `grit` 层级的 XIA 事件或者 VME 事件
  + 对于 XIA 事件，寻找最贴近的触发
    + 如果找到的触发事件编号小于 `tofill`，即对应的触发事件已经填到 TTree 里面了，这是极小概率事件，直接丢弃该探测器事件。但如果这种情况出现很频繁，可以考虑增大缓存，更改为比 1000 大的数。
    + 如果找到的触发事件编号大于等于 `tofill`，但是小于 `tofill+1000`，说明这个触发事件依然在缓存内，将这个事件填入缓存中。
    + 如果找到的触发事件编号大于等于 `tofill+1000`，说明目前在看的探测器事件已经超出缓存范围了，是时候将缓存填充到 TTree 中，并且更新 `tofill` ，直到当前事件编号在 `tofill` 到 `tofill+1000` 之间。
    + 如此循环往复，直到消耗所有的探测器事件。
  + 对于 VME 事件，寻找对应的触发，如果找不到，舍弃该 VME 事件，否则填入 TTree 中。

虽然 XIA 和 VME 的事件的处理流程都叫 smelt，但是实际上做的事情有很大区别。
